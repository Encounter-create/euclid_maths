# 欧几里德之门 写作交互原则

本 skill 规定了在数学公众号写作场景下，AI 与用户协作时必须遵守的交互原则。核心流程采用**两阶段门控**设计：第一阶段最大化创意，第二阶段严守规划。

---

## 两阶段写作流程

整个写作过程分为两个不可逾越的阶段。**第一阶段的任何内容在用户明确批准前，不得进入第二阶段执行。**

### 第一阶段：规划阶段（头脑风暴模式）

**目标**：在正式动笔之前，穷尽创意可能性，形成一份用户认可的最佳写作规划。

#### 1.1 确认写作方向

收到用户的大致写作方向后，若指令中存在以下任一情况，**必须追问澄清**，不得自行推断：

- 选题范围模糊（如"写个组合的"——组合极值？生成函数？计数原理？）
- 难度/受众不明确（竞赛生？普通高中生？数学爱好者？）
- 篇幅预期不清晰（精悍短文？系统长文？）
- 与前作关系不明确（独立篇？系列续作？）
- 任何其他存在歧义的表述

追问应一次聚焦一个关键问题，避免一次抛出过多问题。仅在用户明确声明「自由发挥」「你看着办」等授权用语时，才可以不经追问直接进入头脑风暴。

#### 1.2 头脑风暴

在方向确认后，**进入头脑风暴模式**。此模式下：

- **允许越界**：可以提出超出用户最初结构要求的创意方案，如新颖的文章结构、非常规的切入点、跨领域的类比、独特的叙事手法等
- **多方案并行**：对于关键决策点（结构、选材、叙事线），列出至少 2 个可行方案，标注各方案的核心差异和适用场景
- **大胆推荐**：如果你认为某个方案明显更优，明确推荐并说明理由（创新性、教学效果、读者体验等）
- **不设限**：创意阶段不受任何已有模板或分类的约束——即使最终文章落入三大分类之一，规划阶段的思路可以自由延展

选项呈现要求：每个方案用 2-3 句话概括核心差异和亮点，不做冗长展开。若有推荐，用「推荐」标注。

**头脑风暴的产出物（即写作规划）应包含**：

- 文章标题（或备选标题，标注推荐）
- 文章类型（三大分类之一，或新类型建议）
- 结构骨架（各板块及层级）
- 核心内容要点（每板块的关键论述/定理/例题概要，一句话即可）
- 数学史穿插的选题方向（如适用）
- 创新点说明（与过往文章或同类文章的差异点，1-2 句）

#### 1.3 用户确认（门控点）

规划呈现给用户后，**必须等待用户明确批准**。用户可能：

- 直接批准 → 进入第二阶段
- 要求调整某部分 → 修改后重新提交确认
- 否决并要求重新规划 → 回到 1.2

**在获得批准前，绝对禁止进入第二阶段。**

---

### 第二阶段：写作阶段（严格执行模式）

**目标**：严格按照已批准的规划完成文章撰写，零偏差执行。

#### 2.1 执行纪律

- **规划即合同**：已批准的规划是第二阶段的唯一执行依据，不得擅自增删板块、改变结构、调整选材、更换叙事线
- **禁止越权创意**：第二阶段禁止提出任何新的创意方案或结构变更。即使是"更好的想法"，也必须忍住——它们可以记录下来，在文章完成后作为"后续可探讨方向"向用户提及
- **一致性问题除外**：如果在执行过程中发现规划存在内在矛盾（如某板块篇幅与整体比例严重失调），可以提出调整建议，但必须说明原因并等待确认

#### 2.2 写作执行

按 euclids-gate-math-writing 规范撰文，保存到 output 目录，更新索引。

## 交互原则

以下原则在两阶段中全程生效：

### 原则 1：不确定即追问

当用户的指令表述不清、存在歧义，或 AI 无法准确理解用户意图时，**必须暂停并追问确认**，不得自行猜测、推断或决策。

此原则在两阶段均生效。在第一阶段尤为关键——宁可多追问一轮，也不要带着模糊理解进入头脑风暴。

仅在用户明确声明「自由发挥」「你看着办」等授权用语时，才可以不经追问直接执行。

### 原则 2：修改前先备份

在修改任何 skill 文件（SKILL.md）或已发布的文章文件之前，**必须先将原文件备份**。

备份规则：
- 备份文件名格式：`原文件名_v{版本号}.md`，如 `SKILL_v3.md`、`SKILL_v4.md`
- 版本号从已有备份中递增（检查同目录下已有备份的最大版本号 + 1）
- 备份完成后再执行编辑，禁止直接覆盖原文
- 此规则同时适用于 skill 文件（如 SKILL.md）和已发布的公众号文章

## 适用范围

两阶段写作流程和两条交互原则适用于所有触发 `euclids-gate-math-writing` skill 的写作场景。原则 1（不确定即追问）同时适用于非写作场景。

## 头脑风暴模式判断标准

以下信号表明应进入头脑风暴模式：

- 用户给出了大致方向但未限定结构（如"写一篇关于韦达定理的"）
- 用户给出了结构要求但 AI 看到了更优可能（如"按五段式写"，但 AI 认为该主题适合问题驱动型叙事）
- 用户说"你觉得呢""有什么想法""给点建议"
- 任何 AI 认为存在创意提升空间的场景

以下信号表明可以跳过头脑风暴，直接进入第二阶段：

- 用户明确说"不用脑暴，直接按这个结构写"
- 用户已给出完整且详细的提纲（"按这个提纲写，不要改"）
- 文章类型明确对应已有模板且用户未表达探索意愿
