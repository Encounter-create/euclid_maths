## 三、一号战役 · MathTestGen v1.0

### 战前谋划

**圣旨**（`imperial_decree_mathgen.md`，2.9KB）：

```
战略定位：高中数学教师自动出题工具
MVP 范围：10个考点、6个分数段、4级难度、3种题型
核心交互：选参数 → 自动生成 → 预览 → 打印
约束：纯前端单页，零部署，打开即用
```

**军略**（`battle_plan_mathgen.md`，5.7KB）：

```
技术栈：单页 HTML + Tailwind CDN + KaTeX CDN + 前端直调 OpenAI 兼容 API
数据持久化：localStorage
任务拆解：T1~T8，8项任务，预估 22h/3天
风险：API 调用失败 → 4层 JSON 容错解析兜底
```

**皇帝审阅通过** → 开打。

### 军团协同

```
将军派发军令：
  T1-T6 → 前端百夫长 (file-agent sa_19e9393e66d)  首版 1202行/47.9KB
          ↓ 皇帝追加「自然语言描述+4种输出格式」
          续派同一百夫长 (inherit) → 1433行/60.4KB

          续派同一百夫长 (inherit) → 1670行/71.7KB
  T7-T8 → 并行派出 QA 百夫长 + PM 百夫长
```

### 关键决策

| 节点 | 决策 | 指挥链动作 |
|------|------|-----------|
| 用户说"全部加上" | 采纳三个并行追加需求 | 参谋→将军→同一百夫长继续（不换人，保上下文） |
| 用户叫停跳步 | 建立"先审再打"铁律 | 宰相/参谋产出后必须等皇帝审阅 |
| 三次派发同一百夫长 | 使用 `inherit_agent_id` | 百夫长继承前次战斗记忆，无缝增量开发 |

### 战役伤亡（Bug 记录）

| # | Bug | 根因 | 修复 | 经验 |
|---|-----|------|------|------|
| 1 | 中考/高考模板按钮无法切换 | `selectTemplate` 先设 `S.mode` 再调 `switchMode`，`switchMode` 检测到 mode 未变直接返回 | 调换执行顺序：先 `switchMode` 后覆写参数 | 状态变更函数需注意调用顺序 |
| 2 | 难度徽章配色全部失效 | JS 返回英文类名但 CSS 只定义了中文类名 | 新增英文类名 CSS 规则 | 前后端命名约定必须统一校验 |
| 3 | 高级模式按钮根本点不动 | `''` 未转义导致 JS 解析器提前闭合字符串，整个 `<script>` 块解析失败 | `''` → `\x27` hex 转义 | 拼接 HTML 属性中的 JS 字符串需严格转义 |

# 作战经验总结 & 工作流优化提案

> 以下提案需皇帝审阅批准后方可生效。批准后将备份原工作流，再执行修改。

## 经验 1：代码产出后缺乏自动化验收

**现象**：v2.0 产出后，Bug 3（`''` 未转义）在代码中沉睡，直到皇帝手动点击才发现。如果有自动化检验环节，可在百夫长交付后立即拦截。

**提案 A：增设"宪兵百夫长"自动化验收环节**

在百夫长交付代码后，将军自动追加一步验收：

| 步骤 | 内容 |
|------|------|
| 1 | 百夫长产出 index.html |
| 2 | 将军自动派 browser-agent 打开页面 |
| 3 | 检查 Console 无 JS 错误 |
| 4 | 自动化交互验证（点击模式切换、选择模板等关键路径） |
| 5 | 通过 → 交付皇帝；不通过 → 退回百夫长修复 |

**影响**：将军多一次 `dispatch_task` 调用，但可避免"交付后发现僵尸代码"。

## 经验 2：bug 修复后缺少回归验证记录

**现象**：修复 Bug 3 后做了验证但没留下记录。后续回归时缺乏参照。

**提案 B：补充"QA 百夫长战后检查"环节**

在 bug 修复后，补派 QA 百夫长做一次回归，更新原有的 qa_report.md，记录：
- 修复项及验证结果
- 回归未破坏的已有功能

**影响**：每次 bug 修复后多一次 QA 派发，约 1min。

## 经验 3：多轮增量开发时上下文膨胀

**现象**：一号战役中三次增量追加都继承同一个百夫长实例，上下文越来越大，可能影响效率。

**提案 C：增量超过 2 轮时要求将军出具"增量摘要"**

当同一百夫长被继承超过 2 次时，将军在新军令中附带前两轮的战报摘要，让百夫长不必回溯全部历史。

**影响**：将军编写成本轻微上升，百夫长执行效率提升。
