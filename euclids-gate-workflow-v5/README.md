| Skill | 角色 | 触发场景 |
|-------|------|---------|
| **euclids-gate-manage** | 调度中枢：双系统（自动/手动）协调、索引读写、文件保存 | 写文章、修改文章、文章管理 |
| **euclids-gate-math-writing** | 写作引擎：精神纲领、写作宗旨、三大文章分类模板、语言规范 | 撰写数学公众号文章正文 |

## 三大文章类型

| 类型 | 典范 | 核心结构 |
|------|------|---------|
| **1a. 理论专题文（定理运用）** | No1 鸽巢原理、No7 柯西、No8 母函数 | 问题驱动或证明驱动 → 实例递进 → 启发 |
| **1b. 理论专题文（一般方法）** | No4 组合极值、No3 网格作图 | 定义问题类型 → 经典案例 → 方法框架 |
| **2. 典型例题剖析文** | 一道恒等式三把钥匙 | 多法详录 → 方法归一 → 推广 |
| **3. 习题总解** | No10 前九篇课后习题全解 | Part1 习题集 + Part2 解答 + Part3 方法地图 |

## 写作宗旨：何以所以然

所有文章的根本目标：让学生不仅知道「怎么做」，更理解「为什么可以这么做」。定理或方法的呈现必须遵循认知的自然顺序，而非教科书式的机械排列。

## 精神纲领（六条）

1. **人是终极目的** — 以数学启发心灵，以几何强大思维
2. **教育本质：点燃火焰** — 唤醒思考，而非灌输知识（苏格拉底/雅斯贝尔斯）
3. **数学美学** — 追求优雅、精彩、和谐的结构之美
4. **热爱即天赋** — 猴子有猴子的森林，鱼有鱼的大海
5. **数学即心性修炼** — 专注、纯粹、判断何为美
6. **社会关怀与师者修养** — 让优质数学思维触手可及

## 工作流：手动系统（两阶段门控）

```
用户发出写作指令
    │
    ▼
══════════════ 第一阶段：规划阶段 ══════════════
    │
    ▼
[manage] Step 1: ask_user — 本期有方向还是自由选题？
    │
    ▼
[interaction] Step 2: 头脑风暴模式
    │   ├─ Phase 1 头脑风暴：只出内容（中心思想、叙事主线、知识点清单）
    │   │   禁止设计章节结构、命名小节、编排序号
    │   ├─ 提交规划（标题/类型/结构/要点/创新点）
    │   └─ 等待用户批准 ← 【门控点，不可逾越】
    │
    ▼ 批准后方可进入
══════════════ 第二阶段：写作阶段 ══════════════
    │
    ▼
[interaction] Step 2.5: read_text 重读 SKILL.md，逐条对照模板自检
    │
    ▼
[math-writing] Step 3: 严格按模板骨架列提纲 → 写正文 → 保存到 output/
    │   ├─ Phase 2 列提纲：确定主题→确定类型→对照模板列提纲（骨架顺序和模块
    │   │   边界不可变，作为公众号格式标识）
    │   └─ 按提纲写作 → write_file 保存 → 检查
    │
    ▼
[manage] Step 4: 更新 article_index.json + agenda
    │
    ▼
[manage] Step 5: yyb-product 声明产出物
```

## 安装方法

### 方式一：导入到 Marvis/其他 Agent

将以下四个目录复制到你的 Agent 的 skills 安装目录（通常为 `skills/market/`）：

```
euclids-gate-agenda/
euclids-gate-manage/
euclids-gate-math-writing/
euclids-gate-writing-interaction/
```

然后刷新 Agent 的 skill 列表。

### 方式二：手动加载

在对话中逐条告诉 Agent：加载以上四个 skill 的 SKILL.md 内容作为执行规范。

### 配套文件

将 `article_index.json` 放置于你的项目 output 目录中，或修改 manage skill 中的索引路径。

## 语言规范速查

- 中英混排：术语/定理名/人名用英文，论述用中文
- 第一人称复数「我们」贯穿全文
- 学术 + 口语混合，允许自然的思考流露
- 禁止表情符号
- 名言引用中英双语，标注出处
- 数学公式用 LaTeX（`$...$` 行内 / `$$...$$` 块级）

## 文件版本管理

- 修改 SKILL.md 前必须备份为 `SKILL_v{N}.md`
- 修改已发布文章前必须保留旧版本为 `No{id}_主题_v{N}.md`
- 版本号递增，不覆盖历史
