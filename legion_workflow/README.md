# 罗马军团开发体系 (Roman Legion Development System)

> 一套基于多层 AI Agent 协作的软件开发工作流框架。
> 把一人公司拆成一支军队——皇帝下令，宰相谋划，参谋定策，将军指挥，百夫长冲锋。

## 这是什么

当你用 AI 做软件开发时，最大的问题不是 AI 不够聪明，而是**你指挥不动它**——需求越复杂，AI 越容易迷失上下文，产出的代码越凌乱。

罗马军团解决的就是这个问题。它把一人公司的所有角色拆成层层封装的 AI Agent，形成六层金字塔指挥链：

```
皇帝 (你)
  │  "我要一个记账小程序"
  ▼
宰相 AI                      需求分析 → 圣旨
  │  "目标用户：个人用户，MVP 功能：记账/分类/统计"
  ▼
参谋 AI                      技术选型 → 军略
  │  "用 React + IndexedDB，拆为 8 个任务"
  ▼
将军 AI                      任务调度 → 军令
  │  "前端百夫长，接 T1~T3，归营时限 3 天后"
  ▼
百夫长 AI × 5                专业执行 → 战报
  │  前端 / 后端 / QA / UIUX / PM
  ▼
Sub Agents                   文件操作 / 浏览器 / Shell / 搜索
```

每一层只做自己该做的事，不该问的不问，不该管的不乱管。

## 核心法则（10 条军规）

### 战前——不打无准备之仗

| # | 法则 | 一句话 |
|---|------|--------|
| 1 | **战前谋划不可越** | 圣旨 + 军略 → 你审阅 → 才发兵。禁止跳步直接写代码。 |
| 2 | **逐级抽象** | 你只管"要什么"，百夫长只管"怎么实现"。上下级不越界指挥。 |

### 战中——令行禁止

| # | 法则 | 一句话 |
|---|------|--------|
| 3 | **百夫长继承制** | 同一战线的增量任务不换人，保持上下文连续。 |
| 4 | **并行不悖** | 无依赖的任务同时派出，不排队等。 |
| 5 | **增量摘要机制** | 同一百夫长超过 2 轮时，新军令附带前两轮摘要，防上下文膨胀。 |
| 6 | **军令状不可改** | 发兵后追需求 → 另起战役，不塞进当前军令。 |

### 战后——眼见为实

| # | 法则 | 一句话 |
|---|------|--------|
| 7 | **侦察兵先行** | 交付前截图验证，眼见为实。 |
| 8 | **宪兵自动化验收** | 代码交付自动检查 Console 错误 → 验证交互路径 → 通过才放行。 |
| 9 | **战损必复盘** | Bug 必须定位根因 → 修复 → QA 回归检查。 |
| 10 | **战后必修史** | 每次战役写复盘，提优化提案到 IMPROVEMENTS.md，迭代工作流。 |

## 协同作战流程

```
1. 你（皇帝）下达项目需求
2. 判断规模 → 选择完整战役 或 快速战役模式
3. 确认阵型 → 宰相产出圣旨（需求文档）
4. 参谋产出军略（技术方案）
5. ★ 你审阅 ★ → 批准后发兵
6. 将军 → 军略拆为军令，按阵型分派百夫长
7. 百夫长 → 执行战前自检 → 调用 Sub Agent → 交付前自检 → 提交战报
8. 将军 → 汇总战报，宪兵验收 + 侦察兵截图
9. 交付
10. 战后复盘 → 优化提案写入 IMPROVEMENTS.md
```

## 快速开始

### 准备工作

1. 确保你使用的 AI 助手支持 Skill 加载机制（如 Claude Code、Marvis 等）
2. 运行安装脚本：`.\install.ps1 -Target claude` 或 `.\install.ps1 -Target marvis`

### 启动军团（完整战役）

对 AI 助手说：

> 启动军团，帮我做一个 [你的项目描述]

AI 将自动按军团流程执行：
1. 选择阵型 → 以**宰相**身份分析需求，产出圣旨
2. 以**参谋**身份设计方案，产出军略
3. 提交你审阅
4. 以**将军**身份分派百夫长执行
5. 宪兵+侦察兵验收后交付

### 快速战役（小项目）

> 快速战役，帮我在 MathTestGen 上加一个导出 PDF 功能

三合一谋划（圣旨+军略+军令合并），你审阅一次即可发兵。

### 单独使用某个角色

你也可以单独加载某个角色 Skill，跳过完整流程：

- **只需需求文档**：加载 `legion-chancellor`，说"帮我分析这个项目的需求"
- **只做技术方案**：加载 `legion-advisor`，说"帮我做这个项目的技术选型"
- **只需任务管理**：加载 `legion-general`，说"帮我把这些任务分派出去"

## 设计哲学

1. **一人成军**——单人开发者借助 AI 获得完整团队的能力
2. **上下文隔离**——每个角色只知道它需要知道的，不会被无关信息淹没
3. **逐级抽象**——皇帝看圣旨，百夫长看军令，信息在每一层被精炼
4. **强制审阅**——发兵前必须皇帝批准，AI 不会擅自行动
5. **自动化验收**——宪兵 + 侦察兵代替人工检查，交付即合格
6. **复盘驱动进化**——每战必写战史，提案追踪到 IMPROVEMENTS.md，工作流持续迭代
